# 워크플로우: 이슈 노트 업데이트

**목표**: 작업 완료 후 이슈 노트에 작업 로그를 추가하고, 연관 이슈를 링크하며, 메인 TODO의 체크박스를 업데이트한다.

## 경로 정보

- **TODO 파일**: `/Users/choegihwan/Documents/Projects/Obsidian-frontend-journey/Project/Resume with AI/개선 TODO.md`
- **이슈 노트 디렉토리**: `/Users/choegihwan/Documents/Projects/Obsidian-frontend-journey/Project/Resume with AI/`
- **프로젝트 루트**: `/Users/choegihwan/Documents/Projects/resume-with-ai/web/`

## 1단계: 대상 이슈 확인

- 인자로 이슈 제목이 주어진 경우: 해당 이슈 노트를 찾는다.
- 인자가 없는 경우: 대화 컨텍스트에서 현재 세션에서 작업한 이슈를 추론한다.
- 추론이 불가능하면 `AskUserQuestion`으로 어떤 이슈를 업데이트할지 물어본다.

이슈 노트 디렉토리에서 해당 `.md` 파일을 읽는다.

## 2단계: 작업 내용 분석

대화 컨텍스트와 git 상태를 분석하여 이번 세션에서 수행한 작업을 정리한다.

```bash
git diff --stat
git log --oneline -5
```

분석 항목:
- 변경/생성된 파일 목록
- 구현한 내용 요약
- 발견된 이슈나 추가 작업 사항
- 완료 여부 판단

## 3단계: 이슈 노트에 작업 로그 추가

이슈 노트의 `## 작업 로그` 섹션에 내용을 추가한다. 섹션이 없으면 파일 끝에 생성한다.

### 작업 로그 형식

```markdown
## 작업 로그

### {YYYY-MM-DD}
**상태**: {완료 / 진행 중 / 보류}

**작업 내용**:
- {구체적으로 한 일 1}
- {구체적으로 한 일 2}

**변경 파일**:
- `{파일경로1}` - {변경 내용 요약}
- `{파일경로2}` - {변경 내용 요약}

**발견 사항**:
- {작업 중 발견한 것, 없으면 생략}
```

기존 작업 로그가 있으면 그 아래에 새 로그를 추가한다.

## 4단계: 연관 이슈 링크

작업 중 연관성이 발견된 다른 이슈가 있으면, 이슈 노트에 `## 연관 이슈` 섹션을 추가/업데이트한다.

### 연관 이슈 판단 기준
- 같은 파일을 수정하는 이슈
- 선행/후행 관계가 있는 이슈 (예: Rate Limiting → 입력 검증)
- 작업 중 발견된 추가 개선점이 다른 이슈에 해당

### 형식

```markdown
## 연관 이슈
- [[{연관 이슈 제목}]] - {관계 설명}
```

연관 이슈의 노트에도 역방향 링크를 추가한다 (양방향 링크).

## 5단계: TODO 체크박스 업데이트

`개선 TODO.md`를 읽어서 해당 이슈의 체크박스를 업데이트한다.

### 판정 기준

| 상태 | 조건 | 변경 |
|------|------|------|
| 완료 | 구현 완료 + 동작 확인 | `- [ ]` → `- [x]` |
| 미완료 | 부분 완료 또는 추가 작업 필요 | 변경 없음 |

- **불확실한 경우** 사용자에게 확인을 요청한다. 임의로 완료 처리하지 않는다.

## 6단계: 결과 보고

업데이트 결과를 사용자에게 보고한다:

```
## ✅ 이슈 업데이트 완료: {이슈 제목}

**상태**: {완료 / 진행 중}
**작업 로그**: 추가됨
**연관 이슈**: {연관 이슈 목록 또는 "없음"}
**TODO**: {체크 완료 / 미변경}

{완료된 경우: 다음 이슈를 진행하려면 `/issue-start`를 사용하세요.}
{미완료인 경우: 남은 작업 요약}
```
