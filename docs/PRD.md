## 웹 이력서 및 포트폴리오 분리 기획서

### 1. 핵심 전략

* **이력서:** 후킹과 요약. 채용 담당자가 1분 안에 지원자의 가치를 파악할 수 있도록 숫자로 증명된 비즈니스 임팩트와 핵심 키워드만 남깁니다.
* **포트폴리오:** 증명과 스토리텔링. 이력서에서 호기심을 느낀 면접관이 지원자의 깊이 있는 기술적 고민, 아키텍처 설계, 문제 해결 과정을 케이스 스터디 형태로 읽을 수 있도록 구성합니다.

### 2. 이력서 기획 및 UI/UX 설계

**콘텐츠 구성**

* **헤더 및 요약:** 이름, 연락처, 링크와 함께 본인을 한 줄로 정의하는 강력한 문장 배치.
* **핵심 역량:** 단순 나열보다는 어떤 영역에서 이 기술들을 활용해 왔는지 도메인이나 목적별로 묶어서 표현.
* **경력 및 프로젝트:** Problem, Action, Result 구조를 과감히 버립니다. 대신 **[어떤 기술을 써서] -> [무엇을 개선/개발하여] -> [어떤 수치적 임팩트를 냈는지]**를 한두 줄의 불릿 포인트로 압축합니다.
* 예시: 장애 인지 시간을 10초에서 3초로 70% 단축하는 모니터링 대시보드 아키텍처 재설계 주도.




* 
**교육 및 기타:** 동의대학교 졸업, 항해 플러스 수료, 공인어학성적 등 핵심 내용만 간결하게 배치.



**UI/UX 설계 포인트**

* **F자형 시선 이동 고려:** 채용 담당자는 위에서 아래로, 왼쪽에서 오른쪽으로 훑어봅니다. 좌측 1단 레이아웃이나, 텍스트가 넓게 퍼지지 않도록 최대 너비를 제한하여 가독성을 높이는 것이 좋습니다.
* **여백의 미:** 텍스트가 빽빽해 보이지 않도록 줄 간격과 단락 간격을 넉넉하게 줍니다.
* **연결 고리 버튼:** 각 주요 프로젝트 설명 끝에 '상세 문제 해결 과정 보기' 또는 '포트폴리오 읽기' 같은 직관적인 버튼이나 링크를 배치하여 자연스럽게 포트폴리오로 유도합니다.

### 3. 포트폴리오 기획 및 UI/UX 설계

**콘텐츠 구성 (스토리텔링 중심)**
포트폴리오는 블로그 글이나 기술 케이스 스터디처럼 읽히도록 구성합니다.

* **프로젝트 개요:** 프로젝트의 목적, 팀 규모, 본인의 역할, 사용 기술 스택 요약.
* **배경 및 문제 상황:** 왜 이 아키텍처 개편이 필요했는지, 기존 방식의 한계는 무엇이었는지 서술합니다. 예를 들어 DOM 노드가 23,000개 이상 증가해 리플로우 비용이 커졌던 상황 등을 구체적으로 적어줍니다.


* **기술적 의사결정 및 해결 과정:** Action에 해당하는 부분입니다. 단순히 '무엇을 했다'가 아니라, '왜 A 대신 B를 선택했는지' 트레이드오프를 고민한 흔적을 담습니다.
* **결과 및 배운 점:** 수치적인 임팩트와 더불어, 이 과정을 통해 엔지니어로서 어떤 인사이트를 얻었는지 회고를 덧붙입니다.

**UI/UX 설계 포인트**

* **풍부한 시각 자료:** 아키텍처 다이어그램, 개선 전후의 성능 비교 그래프, 또는 핵심 코드 스니펫 등을 활용해 긴 글의 지루함을 덜어줍니다.
* **네비게이션 사이드바:** 내용이 길어질 수 있으므로, 화면 좌측이나 우측에 목차를 고정해 두어 원하는 프로젝트나 섹션으로 쉽게 이동할 수 있게 합니다.
* **PDF 출력 최적화:** 브라우저의 인쇄 기능을 사용할 때 레이아웃이 깨지지 않도록 프린트용 스타일을 반드시 고려해야 합니다.
* 불필요한 웹 전용 UI 요소 숨기기.
* 페이지가 넘어갈 때 중요한 문단이나 이미지가 중간에 잘리지 않도록 페이지 넘김 방지 처리.
* 링크 텍스트 옆에 실제 URL이 인쇄되도록 처리하는 것도 좋은 디테일입니다.



### 4. 사용자 흐름 시나리오

1. 면접관이 웹사이트 접속 후 **이력서** 페이지를 봅니다.
2. 깔끔한 UI와 핵심 숫자로 정리된 요약본을 1~2분 내외로 스캐닝합니다.
3. '데이터 그리드 성능 개선' 항목의 요약된 임팩트를 보고 기술적 디테일이 궁금해집니다.
4. 항목 바로 아래에 있는 **[상세 케이스 스터디 보기]** 버튼을 클릭합니다.
5. **포트폴리오** 페이지의 해당 프로젝트 섹션으로 부드럽게 이동하여, 깊이 있는 문제 해결 스토리를 읽게 됩니다.
6. 필요시 포트폴리오 페이지에서 브라우저 인쇄 기능을 사용해 PDF로 저장합니다.
