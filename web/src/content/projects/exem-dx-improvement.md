#### CI/CD Report Hub (Faster Test Feedback Loop)

**Problem**

* 아티팩트 확인을 위해 대용량 파일을 매번 내려받아야 해, 테스트 피드백 루프가 지연되고 검증 시작 자체가 늦어졌습니다.

**Action**

* 다운로드 없이 결과를 즉시 확인하는 브라우저 스트리밍 뷰어를 구축해, 리포트 확인 단계를 접근 중심 흐름으로 전환했습니다.
* 리포트 업로드를 CLI 한 줄로 표준화하고 저장 메타데이터 구조를 단순화해, 운영 부담을 늘리지 않고 자동화를 정착시켰습니다.

**Result**

* 테스트 결과 접근 시간이 3분에서 5초 내외로 97% 개선되어, 검증 착수와 피드백 반영 속도가 크게 빨라졌습니다 (Cut access time by 97%).
* 다운로드 대기 중심의 비효율을 제거해 팀 전체의 반복 검증 리드타임을 안정적으로 단축했습니다.

#### GitLab AI Ops (In-context AI Review Workflow)

**Problem**

* 코드 리뷰 중 AI 도움을 받기 위해 외부 도구로 이동하는 과정이 반복되어, 리뷰 집중도와 처리 속도가 함께 저하됐습니다.

**Action**

* MR 댓글에서 바로 실행되는 명령형 AI 워크플로를 도입해, 질의응답·본문 작성·변경 시각화를 리뷰 맥락 안에서 처리하도록 구성했습니다.
* 팀이 GitLab 화면을 벗어나지 않고 협업하도록 운영 규칙을 통일해, 기능 사용 진입 장벽과 문맥 전환 비용을 낮췄습니다.

**Result**

* 문맥 전환 없이 AI를 활용하는 리뷰 흐름이 정착되어, 리뷰 품질과 작업 속도 모두 가시적으로 개선됐습니다.
* 리뷰 과정의 반복 템플릿 작업이 줄어, 핵심 설계 검토에 집중할 수 있는 시간이 늘었습니다.

#### Legacy Tooling (Legacy Debugging & Onboarding)

**Problem**

* 레거시 구조와 전역 의존으로 디버깅 난도가 높았고, 신규 인원의 로컬 환경 구성에 반나절 이상 소요됐습니다.

**Action**

* 전역 상태 추적·SQL 로그 확인·컴포넌트 구조 시각화를 제공하는 디버깅 도구를 구축해, 분석 과정을 관측 가능한 흐름으로 전환했습니다.
* 분산 소스와 복잡한 설정 절차를 통합한 자동화 CLI를 제공해, 초기 환경 세팅의 반복 수작업을 제거했습니다.

**Result**

* 신규 입사자 환경 세팅 시간이 수 시간에서 5분으로 단축되어 온보딩 초기 생산성이 빠르게 올라갔습니다 (Reduced setup time to 5 minutes).
* 상태 추적 기반 분석이 가능해져 레거시 트러블슈팅 시간이 줄고, 문제 재현과 원인 파악의 일관성이 높아졌습니다.
