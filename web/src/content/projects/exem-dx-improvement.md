#### CI/CD Report Hub

<h3 id="problem">Problem</h3>

* 폐쇄망 환경에서 아티팩트(커버리지·보안·VRT 리포트) 확인을 위해 대용량 파일을 매번 내려받아야 해, 테스트 피드백 루프가 지연되고 검증 시작 자체가 늦어졌습니다.

<h3 id="decision">Decision</h3>

* CLI 업로드 → Nest.js API → MinIO 저장 → 정적 서빙 구조로 브라우저 스트리밍 뷰어를 구축해, 다운로드 없이 결과를 즉시 확인할 수 있도록 전환했습니다.
* HTML 리포트라면 종류를 가리지 않고 업로드·열람할 수 있도록 설계해, 운영 부담을 늘리지 않고 자동화를 정착시켰습니다.

<h3 id="result">Result</h3>

* 테스트 결과 접근 시간이 3분에서 5초 내외로 97% 개선되어, 검증 착수와 피드백 반영 속도가 크게 빨라졌습니다.
* 다운로드 대기 중심의 비효율을 제거해 팀 전체의 반복 검증 리드타임을 안정적으로 단축했습니다.

#### GitLab AI Ops

**Problem**

* 폐쇄망·보안 제약으로 외부 SaaS AI 도구를 사용할 수 없어, 코드 리뷰 중 AI 도움을 받기 위해 별도 환경으로 이동하는 과정이 반복되며 리뷰 집중도와 처리 속도가 함께 저하됐습니다.

**Action**

* GitLab Webhook + N8N 기반으로 MR diff 컨텍스트를 수집해, 댓글 명령으로 질의응답·본문 작성·변경 시각화 결과를 MR 코멘트로 반환하는 워크플로를 구축했습니다.
* 팀이 GitLab 화면을 벗어나지 않고 협업하도록 운영 규칙을 통일해, 기능 사용 진입 장벽과 문맥 전환 비용을 낮췄습니다.

**Result**

* 외부 도구 이동 0회 — 모든 AI 보조 작업을 GitLab 내에서 처리해, 리뷰 시 문맥 전환을 완전히 제거했습니다.
* 반복 템플릿 작업(MR 요약·리뷰 체크리스트·변경 요약)이 자동화되어, 핵심 설계 검토에 집중할 수 있는 시간이 늘었습니다.

#### Legacy Tooling

**Problem**

* 폐쇄망 레거시 구조와 전역 의존으로 디버깅 난도가 높았고, 신규 인원의 로컬 환경 구성에 반나절 이상 소요됐습니다.

**Action**

* 전역 상태 추적·SQL 로그 확인·컴포넌트 구조 시각화를 제공하는 디버깅 도구를 구축해, 분석 과정을 관측 가능한 흐름으로 전환했습니다.
* 분산 소스와 복잡한 설정 절차를 통합한 자동화 CLI(의존성 설치·환경 설정 생성·로컬 서버 실행)를 제공해, 초기 환경 세팅의 반복 수작업을 제거했습니다.

**Result**

* 신규 입사자 환경 세팅 시간이 수 시간에서 5분으로 단축되어 온보딩 초기 생산성이 빠르게 올라갔습니다.
* 상태 추적 기반 분석이 가능해져 레거시 트러블슈팅 시간이 줄고, 문제 재현과 원인 파악이 체계적으로 이루어지게 됐습니다.

<h3 id="retrospective">Retrospective</h3>

* 폐쇄망/레거시 제약 환경에서는 단일 도구 도입보다, 팀이 반복하는 병목을 표준 워크플로로 치환하는 자동화 설계가 실질적인 생산성 개선으로 이어졌습니다.
* 빠른 도입을 위해 운영 진입 장벽을 낮추되, 디버깅 가시성과 결과 검증 흐름을 함께 설계해야 자동화가 일회성으로 끝나지 않는다는 점을 확인했습니다.
