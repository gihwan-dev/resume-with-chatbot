#### CI/CD Report Hub

**Problem**

* 기존 GitLab Artifacts의 물리적 다운로드 방식에 따른 **피드백 루프 지연**.
* 테스트 결과를 확인하기 위해 매번 **수백 MB의 파일을 다운로드**해야 하는 비효율 발생.

**Action**

* Zip 파일 내부 리소스를 다운로드 없이 브라우저에서 즉시 확인할 수 있는 **스트리밍 뷰어** 구축.
* GitLab Runner에서 생성된 리포트를 **CLI 명령어** 한 줄로 업로드할 수 있도록 자동화 도구 제공.
* 별도의 DB 도입 대신 파일 시스템과 아티팩트 내 **JSON 메타데이터**만을 활용하여 운영 복잡도 및 인프라 비용 최소화.

**Result**

* 기존 3분에서 **5초 내외**로 접근 시간 97% 개선 및 즉각적인 피드백 루프 형성.

#### GitLab AI Ops

**Problem**
* 코드 리뷰 시 AI 가이드를 받기 위해 외부 도구로 코드를 복사/붙여넣기 해야 하는 **문맥 전환** 비효율 발생.

**Action**
* **N8N**과 **GitLab Webhook**을 연동하여 MR 댓글 내에서 즉시 실행 가능한 명령어 환경 구현.
* **`/ai`**(질의응답), **`/mr`**(MR 본문 자동 생성), **`/graph`**(코드 변경 사항 시각화) 등 리뷰 생산성 향상을 위한 핵심 기능 제공.
* 팀원들이 GitLab 화면을 벗어나지 않고 리뷰 컨텍스트 내에서 AI와 실시간으로 협업할 수 있는 환경 조성.

**Result**
* 문맥 전환 없는 AI 활용으로 **리뷰 품질 및 작업 속도 가시적 향상**.

#### Legacy Tooling

**Problem**
* **ExtJS 기반 시스템**의 복잡한 구조와 전역 객체 의존성으로 인한 **디버깅 난이도 상승**.
* 로컬 개발 환경 세팅에 **반나절 이상 소요**되어 신규 입사자 온보딩 비용 과다 발생.

**Action**
* `postMessage` 통신을 활용하여 **실시간 ExtJS 전역 객체 상태 변경 추적**, SQL 실행 로그 추적, 컴포넌트 트리 시각화 기능 구현.
* 분산된 소스 코드를 **Git Submodule**로 통합하고, 대화형 CLI를 통해 복잡한 환경 설정을 자동화.

**Result**
* 신규 입사자 환경 세팅 시간을 **수 시간에서 5분으로 획기적 단축**.
* 실시간 상태 추적 도구를 통한 레거시 코드 분석 및 트러블슈팅 시간 단축.
