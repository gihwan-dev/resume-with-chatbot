> 대시보드 코어 레이아웃·상태 관리·폴링 아키텍처 설계 및 구현 주도

#### 고객 특화 모니터링 대시보드 아키텍처 재설계

<h3 id="problem">Problem</h3>

* Vue 기반 레거시 대시보드에서 폴링 주기·재실행 규칙이 화면별로 분산되어 동일 운영 시나리오에서도 네트워크 동작이 달랐습니다.
* 카드형 UI의 데이터 밀도 한계로 대규모 인스턴스 모니터링 효율이 낮았습니다.
* **300개 이상** 이기종 DB 상태 동시 렌더링 시 **DOM 23,315개**, 레이아웃 재계산 `61.4/sec`, 스타일 재계산 `83.4/sec`가 발생하며 INP가 **280ms**까지 상승했습니다.
* 인터랙션 도중 폴링으로 인한 리렌더링이 발생해 인터랙션 중 버벅임이 발생했습니다.

<h3 id="decision">Decision</h3>

* Vue 레거시 대시보드에 중앙 Polling Manager를 도입해 화면별로 분산된 폴링 등록·해제·재실행 규칙을 통합했고, React 마이그레이션을 진행하며 TanStack Query로 전환해 폴링을 선언적으로 관리하도록 개선했습니다.
* 카드형 화면을 그리드 중심 구조로 전환해 단일 화면 데이터 밀도를 높였습니다.
* Playwright E2E 테스트로 알람 등 핵심 기능의 통합 테스트를 작성해, 동일 스펙을 유지한 채 안전하게 마이그레이션했습니다.
* `useFrozenData`로 인터랙션 중 폴링 데이터를 동결해 리렌더를 차단하고, `useDeferredValue`로 폴링으로 인한 렌더링 우선순위를 낮추었습니다.

<h3 id="result">Result</h3>

* 장애 인지 시간 **10초 → 3초**(70% 단축)로 운영 대응 속도를 개선했습니다.
* 그리드 재구성으로 **DOM 20% 이상 감소**, **인터랙션 지연 73-82% 개선**을 달성했습니다.
* `useFrozenData` + `useDeferredValue` 조합으로 인터랙션 중 렌더 경합을 해소해 조작 응답성을 안정화했습니다.
* Playwright E2E 게이트로 마이그레이션 안전성을 확보하고 릴리즈 안정성을 높였습니다.

<h3 id="retrospective">Retrospective</h3>

* 폴링 정책을 화면 단위로 흩어놓지 않고 선언적 데이터 흐름으로 통합해야, 인터랙션 품질과 운영 일관성을 동시에 유지할 수 있음을 확인했습니다.
* 성능 최적화는 단일 기법보다 아키텍처 재설계와 E2E 회귀 검증을 결합했을 때 안정적으로 유지된다는 점을 팀 표준으로 정리했습니다.
