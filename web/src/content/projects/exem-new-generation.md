#### API 레이어 아키텍처 재설계 및 타입 안정성 강화 (API Layer Redesign & Type Safety)

**[API 어댑터 레이어의 실용적 추상화](https://publish.obsidian.md/gihwan-dev/Exem/03-Knowledge/API/%ED%98%84+API+%EB%AC%B8%EC%A0%9C%EC%A0%90)**

**Problem**

* API 호출과 데이터 정제 로직이 화면 단위로 분산되어, 엔드포인트 변경 시 수정 지점 탐색과 영향 파악 비용이 크게 증가했습니다.
* 불필요한 추상화가 누적되며 단순 명세 변경에도 수정 포인트가 늘어, 유지보수 속도와 안정성이 함께 저하됐습니다.

**Action**

* 컨트롤러 구조와 맞춘 API 전용 레이어 규칙을 도입해 타입·호출·캐시 규약을 한 경로로 정리했습니다.
* 공통 응답 래퍼 정리와 중복 추상화 제거를 통해, 화면 코드가 비즈니스 흐름에 집중하도록 책임 경계를 재배치했습니다.

**Result**

* API 명세 변경 시 수정 범위를 단일 영역으로 격리해 사이드 이펙트를 줄였고, 타입 안정성과 변경 대응 속도를 함께 높였습니다.
* 코드 탐색 경로가 단순해져 신규 API 온보딩과 리뷰 커뮤니케이션 비용이 줄었습니다.

#### 확장 가능한 시각화 시스템을 위한 도메인 기반 폼 아키텍처 전환 (Domain-driven Visualization Forms)

**Problem**

* 기능 중심 폼 구조로 인해 새 차트를 추가할 때 공통 스키마 수정이 반복되어, 변경 영향 범위가 넓고 조건부 렌더링 복잡도가 빠르게 증가했습니다.
* 단일 폼에 규칙이 집중되면서 UI와 스키마의 응집도가 낮아져, 버그 원인 추적과 기능 확장이 비효율적이었습니다.

**Action**

* 시각화 도메인별 독립 구조로 재편해 차트 유형마다 스키마와 UI를 분리하고, 공통 빌딩 블록은 재사용 계층으로 분리했습니다.
* 차트 타입 기반 검증 규칙을 도입해 유효하지 않은 조합을 개발 단계에서 차단하도록 설계했습니다.

**Result**

* 신규 시각화 추가 시 기존 코드 수정 없이 확장 가능한 구조를 확보해 기능 추가 리드타임을 줄였습니다.
* 폼 스키마와 UI 응집도가 높아져 조건부 렌더링 오류 가능성을 낮추고 코드 가독성을 개선했습니다.

#### Zustand 스토어 지역화 및 상태 생명주기 최적화 (Scoped Store Lifecycle)

**[기술 블로그: Zustand 스토어를 지역화하기](https://velog.io/@koreanthuglife/Zustand-%EC%8A%A4%ED%86%A0%EC%96%B4%EB%A5%BC-%EC%A7%80%EC%97%AD%ED%99%94%ED%95%98%EA%B8%B0)**

**Problem**

* 전역 스토어가 화면 생명주기와 분리되어 상태 누수 버그가 반복됐고, 수동 초기화 코드가 누적되며 리렌더링과 유지보수 복잡도가 증가했습니다.
* 초기값 주입 규칙이 화면마다 달라, 스토어 구성 보일러플레이트가 늘고 신규 기능 적용 속도가 떨어졌습니다.

**Action**

* Provider 생명주기에 맞춘 지역 스토어 생성/정리 구조를 도입해, 상태 수명을 화면 수명과 일치시키는 표준 패턴을 적용했습니다.
* 스토어 생성 규칙을 공통 팩토리로 통일해 초기값 주입과 훅 구성을 반복 가능한 템플릿으로 정리했습니다.

**Result**

* 스토어 설정 보일러플레이트를 약 70% 줄여 기능 개발 진입 비용을 낮췄습니다 (Reduced store boilerplate by about 70%).
* 수동 초기화 누락 기반 상태 누수 버그를 0건으로 유지해, 화면 전환 안정성과 디버깅 신뢰도를 높였습니다 (Sustained zero leakage bugs).

#### 다국어(i18n) 시스템 안정성 및 검증 자동화 (i18n Reliability Automation)

**Problem**

* 다국어 키 누락이 런타임에서 발견되어 사용자에게 원문 키가 노출됐고, 수동 확인 중심 검증으로 휴먼 에러가 반복됐습니다.
* 키 검증 시점이 늦어 코드 리뷰 단계에서 단순 오탈자 확인 비용이 누적됐습니다.

**Action**

* 타입 기반 키 검증과 정적 검사 규칙을 도입해, 정의되지 않은 키 사용을 개발·빌드 단계에서 사전 차단했습니다.
* 배포 전 자동 스캔 단계를 추가해 미등록 키를 릴리즈 전에 탐지하는 검증 파이프라인을 구성했습니다.

**Result**

* 다국어 누락으로 인한 런타임 장애를 사전에 차단해 사용자 노출 품질을 안정화했습니다.
* 키 검증이 자동화되어 리뷰 리소스를 핵심 로직 점검에 더 집중할 수 있게 됐습니다.
